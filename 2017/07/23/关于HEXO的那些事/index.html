关于HEXO的那些事

General 概括

  • 大概是一时兴起打起了博客的注意，在队友（ji you）的推荐下，选择了HEXO + Github ，从此踏上了一条不归路，网上教程很多很多，挑了几个比较详细的开始配置，几乎每条命令都会报错，不是教程不支持新hexo，就是缺这少那，error看到崩溃，甚至还有教程写反步骤，让人几近抓狂，历时一个晚上（白天还要在西电上课），我成功搭建了它，为了方便广大盆友和16、17级的小师弟师妹们，不再走我的弯路，故，我总结了这篇HEXO教程。

Prophase 前期

  • 此教程是在Win下实现，虽然ubuntu也可以.但是当时虚拟机跑着东西的
  • 搭建环境准备（包括node.js和git环境，gitHub账户的配置）
  • 下载链接如下：
  • Git
  • Node.js
  • 注册Github，准备仓库
  • 在万网上买一个域名，看自己的喜好买就是了。
  • 注：下载环境的时候别乱下一气，看好自己的电脑按配置来安装

配置仓库和SSH秘钥

  • 注册完成后，登录账号，然后在Github页面的右上方选择New repository进行仓库的创建。

嗯，接着：

  • 然后点击Create repository即可。

生成添加密钥

在终端输入：

ssh-keygen -t rsa -C “你的Github的注册邮箱”

最好一直Enter，生成附加密码是个麻烦，等秘钥生成完毕，会得到两个文件id _rsa和id _rsa .pub，用记事本打开id _rsa .pub，
把里面的所有内容全部复制，然后进入

https://github.com/settings/ssh

然后把复制的内容粘贴到Key中，Title里面的内容随便填，点击Add SSH key按钮即可。

  • git命令框中
  • ssh -T git@github.com
  • 看到警告，输入yes就好
  • 发现回显 ：Hi 你的github名！ You ve successfully authenticated，····就是成功了
  • 其中新的hexo可能是弹框让你键入github的用户名密码，不用客气键入它！
  • 在配置_config.yml最后面加入以下代码：
  • deploy:
  • type: git
  • repo: http://github.com/wapchief/wapchief.github.io.git（此处，在下图）
  • branch: master
  • 准备工作完成。

安装 HEXO

  • 就像打怪一样，装备和环境准备好就可以去打Boss了
  • 但是这个Boss明显不太容易打。。
  • 也许会遇到各种各样的问题，有些能百度出来的简单我就不提了，这里遇到的其他大问题我尽可能都写一写

找个你认为开心的地方建个文件夹

在文件夹中，鼠标右键

点击git bash here

  • 键入命令：npm install -g hexo-cli
  • 继续：npm install hexo --save

  • hexo -v查看是否安装完整

看到这没问题的恭喜你安装成功。

下载主题

git clone https://github.com/wuchong/jacman.git themes/jacman

启用主题

　　修改博客目录D:\hexo_config.yml中的theme属性，将其设置为jacman。

theme: jacman

更新主题

　　在主题更新之前，备份目录下的_config.yml文件，容易崩

cd themes/jacman
git pull origin master

出现这种错误，可能是配置_config.yml出了问题，下面deploy的代码，从ssh换成http协议试试，或者https。

ERROR Deployer not found: git

  • 这个问题最恶心，除了执行下面命令外：
  • npm install hexo-deployer-git --save
  • 还要把_config.yml文件中的deploy:
    type: github改成git，
  • not found：git 就把github改成 git，
  • not found：github 就把git改成github
  • 百试百灵

    如果实在没找到问题，看看环境搭的有问题没

  • node -v
  • npm -v
  • git –version
  • hexo -v

    页面不能正常显示，或者404

  • 检查根目录下的_config.yml文件是否配置正确，注意变量:后要加空格。
